# 二手餐饮设备情报平台 · 问题现状与优化方向

> 目标用户：全国各地收二手厨具的二道贩子（发现闭店/急转/整店货源 → 打包收货 → 拆单转卖）。  
> 产品定位：从「多平台信息聚合」升级为「高意向收货情报台」。

---

## 一、核心结论

当前产品更像**公开信息看板**，还不是**帮用户更快收到高利润货源的情报工具**。

| 维度 | 现状 | 应有方向 |
|------|------|----------|
| 价值单位 | 采集条数、平台覆盖 | 可联系线索数、看货率、成交推进率 |
| 产品形态 | 可搜索列表 | 按优先级排序的决策台 |
| 筛选逻辑 | 设备关键词 + 黑名单 | 正负信号评分 + 线索分层 |
| 用户动作 | 浏览、筛选 | 联系、看货、标记无效、复盘 |

---

## 二、问题现状

### 2.1 产品定位与价值

| 问题 | 说明 |
|------|------|
| 离成交太远 | 解决的是「看信息」，不是「值不值得出车联系」 |
| 价值指标错位 | 易用采集量衡量成功，用户关心的是可收货概率 |
| 缺少决策支持 | 无优先级、无推荐理由、无跟进状态 |
| 订阅时机偏早 | 未证明稳定高价值前，「信息库访问权」难收费 |

### 2.2 数据与筛选

| 问题 | 说明 |
|------|------|
| 关键词偏「设备相关」 | 易抓到设备帖，难抓到闭店/整店/急转等收货场景 |
| 噪声高 | 教程、网盘、维修、同行回收、求购、家用、配件等混入 |
| 以黑名单为主 | 缺「闭店、整店、打包、自提」等强正向信号与组合规则 |
| 分类偏设备类目 | 用户更需要「高/中/低意向」而非仅制冷/烹饪 |
| 时效不可靠 | 缺发布时间时易把旧帖当新线索 |
| 去重不完整 | 同货源改标题/跨平台可能重复出现 |
| 缺业务维度标签 | 无同行/求购/服务/家用/整店等区分 |

### 2.3 技术与体验（实现层）

| 问题 | 说明 |
|------|------|
| 后端内存库 | 处理入库后需重启 API 才能看到新数据 |
| 前后端契约 | 分页结构、排序枚举、订阅字段等存在不一致 |
| 展示偏检索页 | 缺「今日精选」「推荐原因」、跟进看板 |

### 2.4 当前采集关键词（参考）

定义于 `collector/keywords.js`，共 9 个：

- 商用冰箱 二手、二手烤箱 餐饮、商用洗碗机 转让  
- 厨房设备低价处理、餐厅倒闭设备、冷柜 二手 餐饮  
- 商用灶台 转让、二手制冰机、餐饮设备 整套转让  

**局限**：偏单品设备搜索，对「闭店清仓、整店打包、桌椅成套」覆盖不足。

## 三、优化方向

### 3.1 产品定义（最高优先级）

1. **收缩范围**：先聚焦一个城市/品类/试点用户，每天输出少量高价值线索。  
2. **改交付物**：从「列表」改为「今日值得联系的 N 条 + 理由」。  
3. **改核心指标**：  
   - 愿意 30 分钟内联系的线索数  
   - 用户原本不会看到的新增线索比例  
   - 高意向线索的联系率 / 看货率  

### 3.2 筛选与过滤

**强正向信号（加权/优先展示）**

倒闭、不干了、急转、清仓、整店、整套、打包、设备清仓、饭店转让、桌椅处理、同城自提、搬迁处理、低价甩卖

**强负向信号（降权或过滤）**

高价回收、上门回收、维修、安装、移机、培训、教程、配方、网盘、自动发货、工厂直销、运损、招商加盟

**组合规则示例**

| 组合 | 处理 |
|------|------|
| 闭店/转让 + 整店/整套 + 冰柜/灶台/桌椅 | 高优先级 |
| 回收/维修 为主且缺出售语义 | 过滤或强降权 |
| 教程/配方 + 网盘/自动发货 | 过滤 |
| 标题含求购、谁有、哪里买 | 标记「疑似求购」，降权 |

**建议新增标签维度**

- 线索类型：整店清仓 / 单品急转 / 同行广告 / 服务维修 / 教程资料 / 疑似求购  
- 意向等级：高 / 中 / 低  
- 物流：同城自提 / 可物流 / 地区不明  
- 货值：大件成套 / 桌椅批量 / 配件碎片  
- 风险：旧帖 / 无联系方式 / 疑似同行 / 疑似家用  

### 3.3 展示与闭环

1. **每条线索展示「推荐原因」**（命中哪些正向/负向信号）。  
2. **跟进状态**：已联系、无效、准备看货、已看货、谈价中、已成交、同行已拿走。  
3. **无效原因**（用于优化规则）：旧帖、不是卖货、已转完、价格太高、太远、非餐饮设备。  
4. **反馈回流**：标记数据反哺关键词与评分权重。

### 3.4 采集策略

1. **补充关键词方向**（示例）：餐厅倒闭 设备、饭店不干了 厨具、整店转让 餐饮设备、后厨设备 打包、店铺清仓 冰柜灶台。  
2. **采集后必跑处理流程**，并保证 API 能读到最新库。  
3. **暂不作为重点**：扩更多平台、复杂权限、花哨可视化。

### 3.5 商业模式（验证后再定）

| 阶段 | 建议 |
|------|------|
| 试用 | 免费或极低门槛，验证线索质量 |
| 验证期 | 低底费 + 成交返点，或按有效线索计费 |
| 稳定后 | 再考虑月费订阅 |

## 四、实施优先级

```text
P0  线索提纯（教程/回收广告/维修/求购/同行/家用/旧帖）
P1  高价值信号评分与排序（闭店/整店/打包/自提）
P2  跟进反馈闭环（状态 + 无效原因 → 规则优化）
P3  试点验证（新增性、联系率、看货率、付费意愿）
P4  商业化与自动化深化
```

---

## 五、一句话

**用户买的不是信息量，而是「更快找到值得出车收的货」。**  
当前最该补的三块：**线索提纯 → 高价值排序 → 成交前反馈闭环**。
